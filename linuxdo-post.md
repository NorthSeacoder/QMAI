# 开源 | 青幕AI写作 —— 一个Vibe Coder花4个月写的长篇小说记忆型AI写作系统

## 前言

先说结论：我不是程序员，4个月前我连一行代码都不会写。但我是一个网文读者+业余写手，一直想用AI辅助写长篇小说，试遍了市面上的工具后发现——**没有一个能真正解决长篇写作的核心问题：AI会遗忘。**

写到第50章，AI忘了第3章埋的伏笔。写到第100章，主角的性格完全变了个人。角色A明明不知道某个秘密，AI却让他说出来了。时间线一团乱麻，设定前后矛盾……

所以我决定自己做一个。

---

## 我是怎么做出来的

全程 **Vibe Coding**。不懂的就问AI，AI写代码，我验证效果，不对就反复调。

4个月，4个大版本迭代。前3个版本基本是在交学费——架构设计错了推倒重来，数据结构不合理全部重构，UI逻辑写崩了从头来过。第4版才算稳定下来，也就是现在开源的这个版本。

说实话，过程很痛苦。没有编程经验意味着你踩的坑别人根本不会踩，花的时间是正常程序员的好几倍。但最终还是做出来了，而且我自己实际在用它写小说，体验比我预期的要好。

**技术栈：** Tauri 2 (Rust后端) + React 19 + TypeScript + Vite + Tailwind CSS + Zustand + Sigma.js

---

## 这个软件解决什么问题

一句话：**让AI在写长篇小说时"记住"之前写过的所有内容。**

| 传统AI写作的问题 | 青幕AI写作的解决方案 |
|---|---|
| 写到后期AI遗忘前文 | 章节快照 + 混合检索 + 上下文包自动组装 |
| 人物性格前后不一致 | 角色状态追踪 + 灵魂绑定 + 人设一致性审查 |
| 时间线混乱 | 结构化时间线 + 连贯性检查 |
| 伏笔丢失或错乱 | 伏笔追踪器 + 债务评分 + 自动检测未回收伏笔 |
| 角色知道不该知道的信息 | 角色认知系统（knows / does_not_know） |
| AI生成的错误内容污染后续 | 草稿/正式章节分离，人工确认后才入库 |
| AI味太重 | 去AI化改写 + 项目灵魂风格控制 |

## 核心功能介绍

### 1. 记忆系统 —— 这是整个软件的灵魂

每章正式保存后，系统自动做一次"章节摄取"，把正文拆解成结构化记忆：

- 章节摘要与结尾钩子
- 出场人物、地点、组织、物品
- 关键事件与状态变化
- 人物关系变化
- 角色认知变化（谁知道什么、谁不知道什么）
- 伏笔新增/推进/回收
- 时间线事件

下次写作时，系统自动把相关记忆组装成"上下文包"喂给AI。AI不再是从零开始写，而是带着完整的前情记忆在续写。

![章节快照](screenshots/snapshot.png)
*▲ 章节快照：每章保存后自动生成的结构化记忆，包含摘要、出场人物、地点等*

### 2. 记忆中心 —— 所有记忆一目了然

记忆中心汇总了整部小说的所有结构化记忆：
- 最近章节快照
- 人物状态追踪
- 角色认知状态
- 伏笔推进情况
- 时间线事件
- 设定事实
- 冲突与矛盾

![记忆中心](screenshots/memory-center.png)
*▲ 记忆中心：按时间顺序查看所有章节的记忆快照，包括人物小传和伏笔计划*

### 3. 角色认知系统 —— 防止角色"开天眼"

这是我最满意的功能之一。系统会追踪每个角色在每个时间点"知道什么"和"不知道什么"：

- **knows**：角色已知信息
- **does_not_know**：角色未知信息  
- **reader_knows_but_character_does_not**：读者知道但角色不知道的（信息差/悬念）

AI写作时会自动参考角色认知，防止角色说出不该知道的话。

![角色认知](screenshots/character-cognition.png)
*▲ 角色认知：追踪每个角色"知道"和"不知道"的信息*

### 4. AI对话写作 —— 带记忆的续写

在AI对话区输入写作指令，系统自动：
1. 读取当前章节细纲
2. 检索相关记忆（关键词 + 向量 + 图谱三路混合检索）
3. 组装上下文包
4. 调用大模型生成

生成的内容显示引用来源，你可以看到AI参考了哪些记忆来写这段内容。

![AI写作](screenshots/ai-writing.png)
*▲ AI对话写作：生成内容带引用来源，可直接保存到章节库*

### 5. 大纲系统 —— 三级大纲管理

支持总大纲 → 分卷大纲 → 章节细纲的三级体系。大纲内容会在写作时自动注入上下文，确保AI不会偏离计划。

还支持"细化生成"功能：基于已有大纲和项目记忆，一次性生成章节细纲、人物小传、组织势力、能力体系、伏笔计划与地点设定。

![大纲系统](screenshots/outline.png)
*▲ 大纲库：支持详细的世界设定，包括地图规则、行进速度等*

![细化生成](screenshots/detail-generate.png)
*▲ 细化生成：基于大纲和记忆，批量生成各类设定文档*

### 6. 角色灵魂系统 —— 让AI写出角色该有的味道

两个层面：
- **项目灵魂**：定义整部小说的叙事气质、语言风格、节奏规则、禁区列表
- **角色灵魂（NvwaSKILL）**：每个角色的说话习惯、思维模式、决策风格

内置了几十个预设角色灵魂范例（秦始皇、李世民、李斯、乔布斯、芒格等），可以绑定到小说人物上，让AI在对话中模仿该角色的表达方式。

![灵魂系统](screenshots/soul.png)
*▲ 角色灵魂：内置预设+自定义，可绑定到小说人物*

### 7. 小说图谱 —— 可视化所有关系

基于Sigma.js的力导向图，自动从章节摄取中构建。节点类型包括人物、地点、组织、物品、事件等，关系类型包括持有、敌对、合作、隐瞒等。

支持过滤器、社区发现、节点类型筛选，直观展示整部小说的关系网络。

![小说图谱](screenshots/graph.png)
*▲ 小说图谱：90/261个页面，280/454条链接，按类型着色的关系网络*

### 8. 审查中心 —— 六维质量把控

提供六个维度的AI审稿：
- 爽感密度
- 设定自洽
- 节奏张力
- 人设一致
- 叙事衔接
- 追读引力

还有事实检查功能，自动检测新章节与既有设定的矛盾，按严重程度分级，提供具体修改建议。

![审查中心](screenshots/review.png)
*▲ 审查中心：事实检查自动检测角色状态变化是否合理*

## 使用流程

1. **新建项目** → 设定小说名、题材
2. **导入/生成大纲** → 世界观、人物设定、分卷计划
3. **配置模型** → 支持OpenAI兼容接口、Ollama等
4. **开始写作** → 在AI对话区输入指令，系统自动组装上下文
5. **审阅草稿** → AI生成的内容默认为草稿，不会污染记忆库
6. **确认保存** → 满意后保存为正式章节，触发自动摄取
7. **查看图谱/记忆** → 随时检视关系网络和记忆状态
8. **AI审稿** → 对写好的章节做质量检查

## 一些使用体验

作为这个软件的作者兼第一个用户，说几点真实感受：

**好的方面：**
- 写到后面确实不会"忘事"了，AI续写时能准确引用前面的设定
- 角色认知系统很有用，再也不用担心角色"开天眼"
- 图谱帮我发现了几个自己都忘了的伏笔
- 审查系统能抓出一些我自己都没注意到的矛盾
- 去AI化改写确实能减少模板感

**需要注意的：**
- 模型质量很重要，垃圾进垃圾出，建议用Claude或GPT-4级别的模型
- 大纲越详细，AI写出来的效果越好
- 章节摄取依赖AI提取，偶尔会有遗漏，建议人工检查快照
- 第一次用需要花一点时间熟悉各个功能的位置

## 技术细节

- **桌面框架**：Tauri 2（Rust后端，体积小性能好）
- **前端**：React 19 + TypeScript + Vite 8 + Tailwind CSS 4
- **状态管理**：Zustand 5
- **图谱**：Sigma.js 3 + Graphology + ForceAtlas2
- **向量存储**：LanceDB
- **混合检索**：关键词(BM25) + 向量(语义) + 图谱(关系)，RRF融合排序
- **数据存储**：全部本地，Markdown + JSON + 向量索引
- **自动更新**：内置Tauri Updater，自动检测GitHub Releases新版本

所有数据存在本地，没有云端依赖（LLM API调用除外）。你的小说内容不会被发送到除你配置的模型服务以外的任何地方。
